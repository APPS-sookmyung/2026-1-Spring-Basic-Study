# 섹션 3: 웹 개발 기초

## 프레임워크 학습 로드맵
Java 기반 백엔드 개발에 입문할 때, 본인의 학습 목적에 따라 **접근 방식**을 다르게 설정해야 함.

* **Spring Boot (입문자 추천)**: 전체적인 서비스 동작 원리를 빠르게 파악하고 실무적인 감각을 익히기 적합
* **Spring (심화)**: 프레임워크의 '날 것(Raw)' 그대로를 경험하며 세부적인 설정과 동작 구조를 깊이 있게 파악하고 싶을 때 학습 적합

### 💡핵심 포인트
Spring은 100% Java로 구현된 프레임워크이므로, 학습 중 어려움을 느낀다면 프레임워크 자체의 문제보다 **Java 언어의 기초 지식(객체지향 등)이 부족한 단계**일 확률이 높으므로 언어 재학습이 병행되어야 함.

## 웹(Web)의 기원과 HTML의 본질
웹은 단순한 기술이 아니라 '정보의 공유'를 목적으로 탄생한 공간임.

### 웹의 시작
* 최초의 웹: 단순 정보 공유용으로 시작
* 가치: 하나의 정보를 전 세계 수많은 사람이 동시에 확인할 수 있는 민주적인 정보 공간임.
  
### HTML (Hyper Text Markup Language)
* 정의: '초능력(Link)'을 가진 문자로 문서의 구조를 나타내는 언어
* 역할: 태그(Tag)를 통해 문장과 단어에 의미를 부여
  * 전 세계 사용자가 동일하게 이해할 수 있도록 웹 콘텐츠의 **구조적 약속**을 정의함.


## 인터넷의 신경망과 통식 규약 (HTTP)
웹이라는 공간이 실제로 존재하고 소통하기 위해서는 물리적인 연결과 약속이 필요함.

### 인터넷(Internet)과 웹(Web)
* **인터넷**: 전 세계적으로 연결된 거대한 신경망(네트워크) 그 자체를 의미
* **웹**: 그 신경망 위에 마련된 정보 공유의 공간

### HTTP (Hyper Text Transfer Protocol)
인터넷 세상을 안전하고 정확하게 돌아다니기 위해 전 세계가 지키기로 한 **전송 규약(Protocol)**

* 비유: 편지를 보낼 때 특정 양식을 지키듯, 인터넷에서도 정해진 규칙 필요
* **주소 체계**: 특정 페이지에 접근하기 위해서는 반드시 정확한 주소(URL)를 알고 있어야 소통이 시작됨
