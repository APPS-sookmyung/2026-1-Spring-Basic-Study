3주차 강의 수강
-웹 개발 초기와 비교하여 현대 웹 개발에서 백엔드와 프론트엔드의 역할이 분리된 이유: 웹 기능이 복잡해짐, 사용자 간 상호작용의 증가로 인해 화면과 데이터 처리를 분리하는 것이 효율적이게 됨
-API:  다른 시스템이나 프론트엔드가 백엔드의 기능이나 데이터에 접근할 수 있게 해주는 '인터페이스' 역할을 하는 것
- 시스템이 자신의 데이터를 외부 애플리케이션이나 사용자에게 제공할 때, 데이터베이스에 대한 직접 접근 대신 API를 사용하는 주된 이유: 내부 시스템 구조를 숨기고, 데이터 접근을 통제하여 보안과 안정성을 확보하기 위함임.
- API(Application Programming Interface)란? :애플리케이션들이 서로 상호작용하기 위해 사용하는 인터페이스
-REST API:  HTTP 원칙을 잘 따르는 API
