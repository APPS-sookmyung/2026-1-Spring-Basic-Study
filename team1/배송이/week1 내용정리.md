프로젝트란?

프로젝트는 하나의 결과를 만들어내기 위한 전체 과정이라고 볼 수 있다.
단순히 거창한 서비스만 의미하는 것이 아니라, 작은 기능 하나를 구현하거나 에러를 해결하는 과정도 모두 프로젝트에 포함된다.

정리하면, 일정 기간 동안 특정 목표를 달성하기 위해 계획하고 실행하는 모든 작업의 흐름이다.

⸻

왜 스프링을 공부할까?

자바로 개발을 하다 보면 반복되는 코드와 복잡한 구조 때문에 개발이 점점 번거로워진다.
이 문제를 해결하기 위해 등장한 것이 바로 스프링이다.

스프링은 자바 기반 개발을 더 효율적으로 만들어주는 도구이며, 개발자가 핵심 로직에 집중할 수 있도록 도와준다.

⸻

프레임워크와 라이브러리

개발을 하다 보면 이미 만들어진 코드들을 활용하게 되는데, 이때 자주 나오는 개념이 프레임워크와 라이브러리다.

프레임워크

프레임워크는 개발의 전체적인 구조와 흐름을 제공하는 틀이다.
이미 필요한 기능들이 어느 정도 구현되어 있어서, 개발자는 그 틀 안에서 코드를 작성하면 된다.

즉, 흐름의 주도권이 프레임워크에 있다.
개발자는 그 구조를 따라가면서 로직만 작성하면 되는 형태다.

라이브러리

라이브러리는 필요한 기능을 가져다 쓰는 도구라고 보면 된다.
개발자가 직접 흐름을 제어하면서, 필요한 순간에 원하는 기능을 호출해서 사용한다.

정리하면,
프레임워크는 “틀 안에서 개발하는 것”
라이브러리는 “필요한 기능만 가져다 쓰는 것”이다.

⸻

스프링 프레임워크

스프링은 자바 개발을 더 편하게 하기 위해 만들어진 프레임워크다.
다양한 기능을 제공하고 확장성이 뛰어나기 때문에 실제 서비스 개발에서 많이 사용된다.

하지만 단점도 존재한다.
	•	설정해야 할 내용이 많고 복잡하다
	•	기능이 많다 보니, 필요한 것만 골라 쓰기 어렵다

특히 웹 개발 수요가 증가하면서 빠르게 개발하는 것이 중요해졌고, 기존 스프링 방식은 다소 번거롭게 느껴지기 시작했다.

⸻

스프링 부트의 등장

이러한 불편함을 해결하기 위해 등장한 것이 스프링 부트다.

스프링 부트는 기존 스프링 프레임워크를 기반으로 하지만,
복잡한 설정을 자동화해서 훨씬 쉽게 개발할 수 있도록 만들어졌다.

주요 특징은 다음과 같다.
	•	대부분의 설정을 자동으로 처리
	•	내장 서버를 제공하여 별도 설정 없이 실행 가능
	•	빠르게 개발 환경을 구성할 수 있음

스프링이 다양한 기능을 제공하는 “기반 프레임워크”라면,
스프링 부트는 이를 더 쉽게 사용할 수 있도록 만든 “편의 도구”라고 볼 수 있다.

정리
	•	프로젝트는 목표 달성을 위한 전체 작업 과정이다
	•	프레임워크는 구조를 제공하고 흐름을 제어한다
	•	라이브러리는 필요한 기능을 가져다 쓰는 방식이다
	•	스프링은 자바 개발을 편하게 만들어주는 프레임워크다
	•	스프링 부트는 복잡한 설정을 줄여 빠른 개발을 가능하게 한다

결국 중요한 것은 단순히 사용하는 것이 아니라,
왜 이런 기술이 등장했는지 흐름과 배경을 이해하는 것이다
