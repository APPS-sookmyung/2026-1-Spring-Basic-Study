#Spring-Basic-Week1

## 프로젝트란?

* 특정한 목표를 세우고, 달성하기 위한 모든 과정들
* 일정 기간 동안 일정한 목적을 달성하기 위한 업무 묶음(프로세스)
* 하나의 프로그램을 완성하는 것, 작은 기능을 만들어내는 것, 에러를 해결하는 것, 기능을 추가하는 것 모두 프로젝트

## 프레임워크란?

* 개발자가 틀 안에서 일하면, 편해진다는 생각에서 출발한 개념
* 필요한 기능들을 미리 구현해놓음
* 다양한 라이브러리까지 준비해놓음
* 이를 통해 개발자는 로직에 집중해서 개발할 수 있게 됨
* 개발을 용이하게 하고 핵심 로직에 집중하게 도와줌

## 라이브러리란?

* 다른 개발자들이 미리 만들어 놓은 유용한 코드의 집합
* 내가 처음부터 끝까지 모든 기능을 만들 필요 없이, 필요한 부품만 가져다 쓰는 방식

## 스프링 프레임워크

* Spring은 Java 프레임워크
* 자바 개발자로서 불편했던 것들, 프로젝트할 때 필요했던 기능들을 해결하기 위해 만들어짐
* 스프링 프레임워크는 사용하려면 많은 설정과 구성을 모두 개발자가 해주어야 함
* 자바 문법을 사용하여 웹을 특화해서 만들고 싶다는 욕구 등장

## 스프링 부트 (Spring Boot)

* 스프링 프레임워크가 자바를 위해 탄생했다면, 스프링 부트는 웹에 특화된 스프링 (그렇다고 웹만 할 수 있는 건 X)
* 스프링 프레임워크를 기반으로 웹에 특화
* 대부분의 기본 설정들이 자동화 (개발자 지향적)
